# 01 口播素材处理（完整流程）

> 从拍摄素材到发布短视频的完整流程。

## 场景描述

你在外面拍了一堆口播视频，回来把素材拷贝到电脑里，放在某个文件夹下。现在想让 AI 帮你从头到尾处理成可以直接发布的短视频。

---

## 第一步：整理素材

把拍好的视频拷到电脑后，告诉 AI 文件在哪里：

```
我拍了一些口播视频，已经拷贝到电脑里了，放在 ~/Videos/2026-04-07/ 这个文件夹下面。
帮我初始化素材库，把这些视频整理好。
```

AI 会自动：
- 创建规范的素材目录结构（`media/raw/`、`media/broll/` 等）
- 把你的视频复制到 `media/raw/` 里
- 扫描并建立素材索引

---

## 第二步：语音识别

```
帮我把 media/raw/ 下面所有的视频都做语音识别，我说的是中文。
识别完以后帮我检查一下转录的文字有没有明显错误，特别是专有名词。
```

AI 会自动：
- 提取每个视频的音频
- 运行语音识别（自动选择最适合你硬件的模型）
- 检测卡壳、口误、填充词（嗯、呃、那个）
- 输出转录文本供你审核

> **提示**：如果有经常被识别错的专有名词，提前告诉 AI，比如：
> `注意这些词不要识别错：Claude Code、OpenClaw、Remotion`

---

## 第三步：选片和剪辑

```
识别结果出来了，帮我看看哪些片段讲得好，哪些是卡壳或者口误需要去掉的。
给每个片段打个分，帮我挑出最好的内容，剪成一个完整的短视频。
要加字幕，竖屏 9:16 格式，适合发抖音。
```

AI 会：
- 给每个片段打 1-5 星的吸引力评分
- 标记需要去掉的卡壳、口误、重复内容
- 推荐最佳片段组合
- 展示方案让你确认
- 确认后渲染最终视频（单次编码，无质量损失）

---

## 一句话搞定版（老手用）

如果你已经熟悉流程，一句话就够了：

```
帮我剪 ~/Videos/2026-04-07/ 下面的口播视频。中文，自动识别，
去掉卡壳和口误，加字幕，竖屏输出，帮我选最好的片段。
```

---

## 调整和修改

如果 AI 给的方案不满意，直接说哪里要改：

```
开头的 hook 不够吸引人，换成片段 7 做开头。
```

```
片段 3 和片段 5 内容重复了，去掉片段 5。
```

```
整体太长了，控制在 60 秒以内。
```

---

## 完整对话示例

### 你：
```
我今天拍了一组口播视频，主题是"程序员副业赚钱"，
素材在 ~/Videos/side-hustle/ 里面，一共 4 个文件。
帮我处理成可以发抖音的短视频。
```

### AI 执行完毕后，你确认选片方案：
```
方案不错，但把片段 7 放到开头做 hook（"你知道程序员下班后做什么最赚钱吗"），
然后接片段 2、4、6、9，最后用片段 12 做结尾。
```

### 你继续追加需求：
```
不错，再帮我：
1. 出一个 1.25 倍速版本
2. 加个轻快的背景音乐，音量 15%
3. 封面标题用"月入过万的 3 个副业"
4. 顺便出一个横屏版本发 B 站
```
